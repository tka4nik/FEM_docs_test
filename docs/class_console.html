◆ AddTag()

void Console::AddTag ( int  id,
std::string  description 
)

Добавление тега для комадны help.

Parameters
id- индетификатор. Каждой команде присваивается любой из добавленных id или id по умолчанию (OtherTag = -1). При вызове команды help все команды сортируются по id (Команды с id по умолчанию выводятся последними)
description- описание id, которое отображается в help.

с помощью этого механизма в команде help можно объединять команды в группы и выводить каждую группу отдельно.

◆ DlgQuestion()

Console::DlgAnswer Console::DlgQuestion ( const std::string &  question,
const std::string &  answ_yes,
const std::string &  answ_no,
bool  loop = false 
)

Задает вопрос пользователю с двумя вариантами ответа. Можно использовать при возникновении некритических ошибок. Например, в случае неудачи тесселяции или триангуляции, можно спросить пользователя, продолжать ли моделирование. DlgQuestion может быть вызвана в любом потоке, в котором есть доступ в Console.

Parameters
question- вопрос, который нужно вывести на экран
answ_yes- утвердительный вариант ответа
answ_no- отрицательный вариант ответа
loop- нужно ли переспрашивать пользователя, если его ответ не совпал ни с answ_yes, ни с answ_no, или сразу вернуть DlgAnswer::Undefined
Returns
вернет DlgAnswer::Yes, если пользователь ввел строку, совпадающую с утвердительным вариантом ответа, и DlgAnswer::No - в случае отрицательного.

◆ Help()

void Console::Help ( const std::vector< std::string > &  args)

Выполняет команду help.

Parameters
args
  1. args - пустой. Выводится отсортированный по тегам список сигнатур команд.
  2. args = {details} Выводится отсортированный по тегам список сигнатур команд с их подробным описанием
  3. args = {<имя_команды>} Выводится информация о сигнатуре команды <имя_команды>
  4. args = {<имя_команды>, details} Выводится информация о сигнатуре команды <имя_команды> и ее подробное описание
  5. args = {notation} - выводится информация о нотации, в которой записаны сигнатуры команд

Функция ввода. Будет постоянно читать поток input_stream, до тех пор, пока не будет введена команда exit.

может выводить информацию в m_output_log(log) и m_message_log(cout). Можно использовать для вывода ошибок. Выводит сообщения цветом ERROR_COLOR (красный)

Parameters
log- указывает в какой именно лог нужно вывести сообщение. Используйте MSGLOG для вывода в m_message_log и OUTPUT для вывода в m_output_log. Для вывода в оба лога передайте MSGLOG | OUTPUT.

Пример использования: ReportError(MSGLOG | OUTPUT, "a = ", 1, "\n b = ", 2.f, EndL); (EndL определен как '
' см. выше)
